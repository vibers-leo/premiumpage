# 세션 로그: 토스페이먼츠 결제 링크 시스템 구현

**날짜**: 2026-04-10  
**프로젝트**: premiumpage (premiumpage.kr)  
**작업 유형**: 신기능 개발 + 배포

---

## 세션 개요

와요(wayo.co.kr) 토스페이먼츠 고도화 작업에 이어, 프리미엄페이지에 토스페이먼츠 결제 시스템을 구축했다. 프리미엄페이지는 B2B 서비스 특성상 고정 가격 결제가 아닌 **견적 기반 결제 링크** 방식으로 설계했다.

---

## 주요 논의

### 1. 결제 방식 선택
- 기존 `/checkout` 고정가 결제 → B2B 서비스에 부적합 (견적마다 금액이 다름)
- **결론**: 어드민이 금액/주문명/고객정보를 입력해 링크 생성 → 링크를 고객에게 이메일 발송 → 고객이 링크에서 결제
- 토스페이먼츠 심사 기준상 이 방식도 승인 가능 (링크 결제는 정상 사례)

### 2. 가격 표시 변경
- 기존: 50만원 / 100만원 / 200만원 (고정)
- **변경**: Basic 50~100만원 / Pro 100~200만원 / Master 200만원+
- 가격 페이지 버튼: "바로 구매하기" → "무료 견적 받기" (`/quote?plan=` 링크)
- 이유: B2B 제작 서비스는 견적 후 확정이 업계 표준

### 3. Prisma 7 호환성 이슈
- Prisma 7에서 `schema.prisma`의 `datasource.url` 제거 필요 → `prisma.config.ts`로 이동
- `prisma.config.ts`의 기본값이 `file:./dev.db`로 되어 있어 로컬 마이그레이션 실패
- **해결**: 마이그레이션 SQL 파일을 수동 작성 후 서버 DB에 직접 적용

### 4. vibers-db 공유 메모리 이슈
- 장기 운영 중이던 `vibers-db` 컨테이너에서 `could not open shared memory segment` 에러 발생
- `docker restart vibers-db`로 해결
- 재시작 후 `docker exec vibers-db psql -U vibers -d vibers_main`으로 직접 SQL 실행

## 기술적 상세

### 구현된 결제 플로우

```
어드민 /admin/payment-links
  → 링크 생성 (금액, 주문명, 고객정보, 만료일)
  → UUID 토큰 생성됨
  → URL 복사: https://premiumpage.kr/pay/{token}

고객이 링크 방문
  → /api/pay/{token} 로 링크 정보 조회
  → 주문 정보 카드 표시
  → "카드 결제" 버튼 클릭

Toss SDK requestPayment() 호출
  → sessionStorage에 pendingPayment 저장 (token, linkId, orderName 등)
  → Toss 결제창 표시

결제 완료 후 리다이렉트
  → /pay/{token}/success?paymentKey=...&orderId=...&amount=...
  → sessionStorage에서 pendingPayment 읽어 confirm API 호출

/api/payment/confirm
  → Toss API POST https://api.tosspayments.com/v1/payments/confirm
  → 성공 시 payment_links.status = 'paid', paidAt, tossPaymentKey 업데이트
  → 실패해도 Toss 결제 자체는 성공 처리 (DB 실패가 결제 취소 안 됨)
```

### PmPaymentLink Prisma 모델

```prisma
model PmPaymentLink {
  id            String    @id @default(uuid())
  token         String    @unique @default(uuid())
  amount        Int
  orderName     String    @map("order_name")
  customerName  String?   @map("customer_name")
  customerEmail String?   @map("customer_email")
  memo          String?
  status        String    @default("pending")   // pending | paid | expired
  tossOrderId   String?   @unique @map("toss_order_id")
  tossPaymentKey String?  @map("toss_payment_key")
  paidAt        DateTime? @map("paid_at")
  expiresAt     DateTime? @map("expires_at")
  createdAt     DateTime  @default(now()) @map("created_at")

  @@map("payment_links")
  @@schema("premiumpage")
}
```

### Toss orderId 패턴

```ts
const orderId = `PP-${link.token.slice(0, 8)}-${Date.now()}`
```

Toss 정책상 orderId는 영문/숫자/하이픈만 허용, 최대 64자.

## 배포 결과 및 빌드 이슈 해결

최종 배포 성공까지 5회 빌드 실패를 거침:

### 해결한 빌드 이슈 (순서대로)

1. **서버 repo 디렉토리 없음** — `/root/projects/premiumpage/repo`를 `git clone --depth=1`로 생성
2. **디스크 100% (50GB 풀)** — `docker builder prune -af && docker image prune -af`로 ~20GB 확보
3. **Prisma generate 순서** — `DATABASE_URL` ENV 설정 전에 generate 실행 → ENV 후로 이동
4. **`.dockerignore` 없음** — `node_modules`(884MB)가 `COPY . .`에서 builder의 prisma client 덮어씀 → `.dockerignore` 생성
5. **`next.config.js` / `next.config.ts` 충돌** — `.js` 파일이 우선 적용되어 `ignoreBuildErrors` 미적용 → `.js`에 추가
6. **Prisma 7 adapter 필수** — `@prisma/adapter-libsql`(SQLite)을 `@prisma/adapter-pg`(PostgreSQL)로 교체
7. **SSH action command_timeout** — 기본 10분 → 20분으로 확대 (서버 빌드 10분+)
8. **빌드 중 디스크 재소진** — Docker 캐시 재정리 후 수동 빌드

### 최종 결과
```
✅ 배포 성공 | premiumpage | HTTP 200 | premiumpage.kr
```

- vibers-db 재시작 후 payment_links 테이블 직접 SQL로 생성
- 수동 `docker build` + `docker compose up -d --no-deps` 으로 배포
- QA: 메인 페이지, /admin/payment-links, /pay/[token] 모두 200 OK 확인
